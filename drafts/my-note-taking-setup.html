For quite some time, Evernote has been a central part of my note-taking workflow and process. I've become pretty comfortable with it, and developed a pretty well worn system for taking and organizing my notes. This post is about all the various kinds of notes and formatting that I've found useful while working with Evernote as my primary note-taking tool.

Goals

A few of my main goals with this system have been:

  • Track all of the useful information I come across and interact with on a daily basis.
    • In particular, I always try to record my reasoning and/or sources of decisions I make so that I can recall why I did made choices I did in the future.
  • Track time spent on various projects for both personal projects, and home/work/consulting.
  • Store and link between topics for easy navigaton.
  • Have a set of contemporaneous notes to help me recall what I did on a given day, and how or why I did it.

By using this system, there have often been times where I'll be trying to recall how or why I did something. Often, I can find the day and notes from that day easily, but I don't always have the one bit of reasoning or exact detail I was looking for, but I'm still able to figure out how or why I did something by getting back into the context of that day and time period by using my notes to help remember. Those occasions often lead me to think more about how to take better notes that capture more detail about the more important considerations I'm making that might be useful in the future.

Overview

There is definitely nothing unique about the way I've been managing my notes in Evernote. Having spent some time becoming more familiar with Org mode now, and having done more research into the TODO list app space, it's become more apparent that everything I'm doing is really common. It's so common that it's pretty much been codified by Org mode and many other similar tools.

I just wanted to document and share my note-taking system here for anyone who's interested so that I can begin to see how it evolves over time as I spend more time researching and working to improve and optimize my system and overall personal knowledge management approach.

Evernote

For my notes stored in Evernote, I've basically settled on three types of files:

  • Daily TODO lists: I usually have at least two or three of these notes running at any given time. I organize them by naming them including the month. For example, right now I have one note that I use to track TODOs for my consulting projects, one for my home projects. These files are used to store just my TODO lists, organized by day, and in the case of consulting, by client as well.
  • Daily Journals: I have a daily journal for more extended notes. I normally organize these by project and month as well.
  • Topical Notes: I create a new topic note any time I have a specific topic that I'd like to add some notes about. Examples of topic notes would be things like "Emacs on OS X", "Cloud Native Engineering", "Yarn and NPM Packaging", "#project MySQL Queries", or "Kubernetes Notes".

Daily TODO Lists

I've established a pretty simple format that I've been using for all of my daily TODO lists. Here what a day normally looks like in my simple Evernote TODO format:

For Wednesday, Oct 16th, 2019

  • Client Name: 9:00am - 10:30am : 10:45am - 4:45pm : 5:30pm - 6:00 : 8.0h
    • Setup and test Jenkins running in Minikube:
      • Setup and configure for app building.
        • Get build pipeline running.
        • Setup and configure to build a docker image on test success.
        • Configure Jenkins to push to container repository. (Docker Hub for now)
    • Sign up for Docker Hub account in order host more than 1 private repository for now.
    • Setup and configure for api building.
    • Look more closely at Elasticsearch logs.

There are a few things going on here:

  • At the top I've got the day of week and date.
  • I have the client name, and every block of time throughout the day that I spent working through the TODO items listed below. At the end of the time blocks list, I have the total hours spent that day for the client in bold.
  • Then, I have the TODO list, organized as a loosely structured dependency tree, as required.

At the beginning of each day, I would simply copy the previous day's TODO list; change the date, delete the time blocks and total hours spent, and remove the completed checkboxes. At the beginning of the next day after the day above, my Thursday, October 17th, 2019 would look like the following:

For Thursday, Oct 17th, 2019

  • Client Name: : 0.0h
    • Setup and configure for api building.
    • Look more closely at Elastic search logs.

As the day goes on, I would add and expand on the above TODO items as necessary to descibe all of the important tasks and deliverables that I worked on, and/or completed over the course of the day.

I always organize these files in reverse chronological order so that the most recent day is always at the top of the file. Sometimes I keep a backlog of TODO items at the top of the file before all of the daily entries, and if that backlog gets long enough I'll probably end up moving it to its own topical note.

Daily Journals

My daily journals are also organized in reverse chronological order like my Daily TODO lists. A typical daily journal entry often complements the associated TODO list. While my TODO list items are short and succinct, my daily journals are where I can save much more depth around the details of what I was doing that day.

Here is an example of a daily journal:

2019-10-19

Elements that should be optimized or done better:

  • Prometheus:
    • Figure out how/if we can alert on webapp/api errors.

  • Jenkins:

    • Builds:
      • Currently, uses a fresh base image e.g. Ruby or Python. Then, installs everything in order to run tests. Then, goes into Docker build and rebuilds the entire image again to ship to image repository.
      • Need to configure max concurrent builds. In order to avoid overloading cluster with Jenkins builds.
        • May not be a problem in practice for quite some time.
    • Authentication:
      • Should probably use SAML for auth at some point. Not sure about this project though.
      • Should use keys directly from Vault through service account role/policy for minimal required secret access.
    • Elasticsearch:
      • Need to only keep last N days of log stash indexes. Need to ensure a process runs to maintain only last N days.

kubernetes-credentials-provider-plugin

After fighting for quite some time with Jenkins resource issues, I started with a fresh Minikube with more memory and cpus was able to run all services concurrently:

minikube start -p mk4 –memory=8g –cpus=4 –bootstrapper=kubeadm –extra-config=kubelet.authentication-token-webhook=true –extra-config=kubelet.authorization-mode=Webhook –extra-config=scheduler.address=0.0.0.0 –extra-config=controller-manager.address=0.0.0.0

Code Snippets, Shell Command, and Queries

Most of my code snippets, and shell commands are either stored alongside their context in my daily journals, or in larger topical files like "#project Setup", or "#project MySQL Queries".

Pinboard Web Bookmarks

I love storing web links to important information in my notes along with the context for why I found them, or where I used them. That means they usually either end up in my Daily Journals or Topical Notes. But, sometimes especially if I'm on my phone, or doing something unrelated and I find something useful, I'll store it as a bookmark using Pinboard. I have the Pinboard bookmarklet setup in Google Chrome on my laptop, and I have PinDroid installed on my Pixel 3a and Pushpin on iPad.

Web Annotations

I recently started using Hypothes.is to easily save annotations from around the web. While I can see the value of annotating pages I'm visiting and I've found it somewhat useful, it has yet to become an important part of my workflow.

Kindle Highlights

I had been thinking about how I'd love to easily keep track of notes from the books I'm reading, but it wasn't until recently while reading the beepb00p.xyz blog that I began to look more closely at how to track my notes digitally by using Kindle highlights.
