While I've been working on upgrading my note-taking process, I started thinking more about general workflow optimization and specifically about what I consider to be a pretty large part of my workflow optimization which is keyboard shortcuts. So, I started this note to catalog and describe some of the most important and perhaps slightly more obscure ways that I'm leveraging keyboard shortcuts throughout my daily digital workflows.

Note: The majority of shortcuts and tools I'm describing are based on my personal setup - a alte 2019 model MacBook Pro running Catalina 10.15.3.

Keyboard Shortcuts ⌨️

Many of the keyboard shortcuts I use most are standard ones like Ctrl-C, Ctrl-V, Command+SPC, etc. I won't go into too much detail about what I consider to be some of the more standard shortcuts, but will focus on detailing a few of what I think are the more eccentric ways that I've attempted to optimize my workflows with keyboard shortcuts.

Fast App Switching

For years, I had simply resigned myself to using the Command+Tab and Command+Shift+Tab to navigate the my open apps. Even though this isn't the worst way to switch apps, the inefficiency of it had been bothering me for quite some time. As I began thinking about the efficiency of my workflow at a more granular level, I decided I should think about fixing this again. I knew there was a faster way to do this, because I'd done it before. My goal was to simply define a few keyboard shortcuts at the OS/App level to help me navigate to my most commonly used applications quickly.

So, I started running Quicksilver again, just to easily setup a few quick app shortcuts for now. I had been using Quicksilver long ago as a way to find and launch apps, before the Mac OS spotlight search became more effective like it is today. I also tried Alfred, but Quicksilver is free while some Alfred features are paid.

Now that I've been using Quicksilver for fast app switching, I've found that this is one of my favorite speed boosts for navigating applications.

Anyway, I currently have just a few key app switching shortcuts that make getting around much easier than continuously using Cmd+Tab and Shift+Cmd+Tab:

  • Cmd+Shift+C: Goto Google Chrome.
  • Cmd+Shift+E: Goto Emacs.
  • Cmd+Shift+I: Goto iTerm. Also need to go in and unmap OS X default shortcut for Notification Center.
  • Cmd+Shift+M: Goto Activity Monitor.
  • Cmd+Shift+X: Goto xCode.

Google Chrome Shortcuts

There are quite a few useful shortcuts that come built into Chrome, and into a few GSuite Apps like Gmail and Docs. I'm always sure to enable keyboard shortcuts for Gmail, and use them heavily in Google Docs.

Tab and Window Navigation

  • Command+N: Open a new window.
  • Command+~: Cycle to next window.
  • Command+Shift+~: Cycle to previous window.
  • Command+Tab: Open a new tab.
  • Command+Shift+{: Move to previous tab (left).
  • Command+Shift+}: Move to next tab (right).

Vimium

Another, recent re-installation for me is the Vimium Google Chrome Extension. Right out of the gate, a few of the most useful features of this extension for me are:

  • j/k: Scroll page up and down.
  • u/d: Page up and down.
  • gg: Scroll to the top of the page.
  • G: Scroll to the bottom of the page.
  • f: Display all link shortcut keys. You can then just type the shortcut key pattern to navigate to the page. Using Shift with the shortcut key pattern opens the link in a new tab.
  • H/L: Navigate back and forward through the tab history.
  • /: Enable in-page text search.
  • ?: Show Vimium help.

Vimium also has its own tab and window navigation commands that mirror those of Vim, but I tend to use the curly brace tab movement commands because of how widespread that functionality is across macOS apps.

There are quite a few other useful features of the Vimium extension, but the above features are the ones that I've found the most useful.

Window Management and Tiling Shortcuts

I've been using Divvy for window/tiling management on macOS for quite some time as well. Out of the box, Divvy binds the Command+D shortcut on macOS, and presents a simple way to visually assign a window to a specific area of the screen. You can also use Divvy to configure your own custom global and Divvy-modal shortcuts to assign the current window to pre-defined screen regions you specify. For example, I've mapped:

  • Command+A: Expand window to fill the screen area.
  • Command+L: Expand window to fill the left side of the screen.
  • Command+R: Expand window to fill the right side of the screen.

Editors

Really, the whole reason that I've used Vim and now Emacs is for the ability to edit text and source code using only the keyboard. Vim is especially useful when interating with servers through the command line, where the stripped down Vi editor is often available. Both Vim and Emacs enable endless keyboard shortcut usage and customization so that you can edit text easily while never having to use a mouse.
