Blog Setup Notes

#emacs #jekyll #orgmode #spacemacs
February 24, 2020
Updated April 20, 2020

I'm currently running this blog using Org mode, Jekyll, and GitHub Pages. I'm using Spacemacs for editing, and Ruby 2.6.5 with Jekyll.

I mostly followed Ian Barton's excellent Using org to Blog with Jekyll post. Having done a bit of static site building in the past, a lot of the simplicity in the initial part of Ian's instructions quickly resonated with me:

Publishing your site with org and Jekyll involves three steps:

  • write your page content using org.
  • use org to export your pages to html in the Jekyll project directory.
  • run Jekyll to convert your html pages exported from org into your final site.

and,

You should setup the directory structure of your org files to mirror that of the Jekyll project. Then when you export your org files as html the files will end up in the correct place in your Jekyll project. I usually place the directory containing my org files in the directory about the Jekyll project directory to make sure that Jekyll doesn't consider .org files to be part of its project.

I also found Junpeng Qiu's Org-mode to GitHub pages with Jekyll post to be a very useful follow on to help resolve what seems to be an issue with an outdated front matter declaration in Ian's post which led to the Jekyll Front Matter YAML being encoded into the HTML and not processed by Jekyll.

It turns out that the correct way to define the YAML front matter in Org mode is: 

#+BEGIN_EXPORT html
---
layout: default
title: Blog Setup Notes
---
#+END_EXPORT

Having fixed the front matter, and re-exported my org mode file content to HTML, this new test page began looking beautiful using the default minima Jekyll theme.

Emacs and Org Directory Structure

My current emacs and Org directory structure is: 

emacs
├── blog # jekyll blog
├── journal # testing org-journal
└── org
    ├── blog # org mode blog posts
    │   ├── _drafts
    │   ├── _notes
    │   ├── _posts
    │   ├── about.org
    │   └── tags.org
    ├── brain # haven't figured out yet
    ├── consulting
    ├── notes
    ├── personal
    ├── work
    ├── notes.org
    └── notes.org_archive

Why Static Site?

  • I wanted to use Org mode to do all of my writing (I'm planning to write more about my inspiration for this setup later).
  • I wanted it to be as fast as possible, and as simple as possible.

Why Jekyll?

  • I wanted to build a simple static site and host on it on GitHub pages.
  • Jekyll is written in Ruby and had 39.8k stars on GitHub at the time of this writing, and I've seen a lot of great sites in Jekyll.
  • I found some reasonable looking documentation about how use Jekyll with Org mode.

Jekyll Setup

I followed these simple Jekyll Quickstart instructions to get going: 

gem install jekyll bundler
jekyll new blog
cd blog
bundle exec jekyll serve

And then, browse to http://localhost:4000.

I placed my Jekyll blog directory into my emacs directory, at the same level as my org mode directory (see above for layout).
