Emacs Packages TODO List

#emacs #orgmode #spacemacs
February 27, 2020
Updated April 19, 2020

I'm still in the process of getting things going and refining my Emacs setup with Spacemacs right now.

Want to Try

  • org-projectile: Manage org-mode TODOs for your projectile projects.
  • org-ql: An Org-mode query language, including search commands and saved views.
  • org-super-agenda: Supercharge your Org daily/weekly agenda by grouping items.

Currently Testing

I'm testing these to see if I can use them in my workflow. I'm not quite sure where they fit yet.

  • magit: It's Magit! A Git porcelain inside Emacs. https://magit.vc.
    • Finding magit to be pretty awesome so far.
  • org-bullets: Show org-mode bullets as UTF-8 characters.
  • org-roam: Rudimentary Roam replica with Org-mode

Currently Using

  • org-mode: A plain text notes and project planning tool for Emacs. There are a lot of elements of org that I'm using and finding extremely useful!
    • Document Structure
      • Headlines
      • Visibility Cycling
      • Motion
      • Sparse Trees
      • Plain Lists
      • Drawers
    • Hyperlinks
      • External Links
        • Super cool thing is deep-linking to headlines in other documents. e.g. file://../filename.org:*headline.
    • TODO Items
    • Tags
    • Dates and Times
    • Refiling and Archiving
      • Refiling
      • Archiving
    • Capture and Attachments
      • Capture
      • Attachments
      • RSS Feeds
    • Agenda Views
    • Markup for Rich Contents
    • Exporting
      • HTML Export
    • Publishing
    • Working with Source Code
      • TODO I'm just scratching the surface here.
    • Miscellaneous
  • helm: Emacs incremental completion and selection narrowing framework https://emacs-helm.github.io/helm/.

Tried and Not Using

  • gnuplot-mode:
    • Looking at and testing Gnuplot again reminded me of what a cryptic syntax it has and how unattractive the generated images are. The fonts and aliasing are pretty bad. I'm sure there must be ways to improve its rendering quality, but I'm not sure it's worth figuring out for me given how unnecessarily complex the gnuplot syntax is, for now.
  • org-journal: A simple org-mode based journaling mode.
    • I've been doing most of my daily journaling in my project notes, so haven't foud as much use for org-journal.
