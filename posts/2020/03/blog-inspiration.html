I've been working on putting this new blog together for a few weeks now. It's taken me a bit longer than I thought it would, but its finally getting to the point where I'm happy with the progress I've made and how it's all coming together. I'm super excited about the underlying Emacs/Spacemacs Org mode setup and the kinds of #pkm hacking that I can see it enabling. I thought it would make a great first post to share a bit about what inspired me to setup this new blog and start blogging again.

A Little Background

I've been doing a lot of research lately, and learning a lot of new things between my work as a software engineering consultant and my own projects.

I'm also pretty serious about how I take notes and organize my personal knowledge database which currently consists of my notes that I'm storing in Evernote, a large collection of webpage bookmarks that I had stored in Pocket and recently moved to Pinboard, and my documents, screenshots and files currently stored in Dropbox (as well as a few other things that I'm using and testing, and planning to document more in the future).

I'm not sure where I fall on the spectrum of note takers and Evernote users, but I've been using Evernote pretty heavily, and have accumulated more than 3,000 notes over the past ten years, often creating more than 20 new notes/month. Sometimes creating more than 40 notes a month.

Before Evernote, and even for some period of time while I began using Evernote I also maintained a daily journal as a text file in Vim. I've been using Vim as my primary source code editor for about 15 years, and have developed quite a fondness for all the keyboard shortcuts and efficiency that mouseless text editing enables.

As my recent workflow has involved more online research and note taking than normal, I've been noticing the efficiency of my note taking system declining due to a lot of context switching associated with working between Google Chrome with multiple windows and tens of browser tabs, multiple concurrent notes in Evernote, and shifting between research and discovery, writing code, testing, and note taking. I'd been noticing these issues bothering me for a while, but hadn't really given too much thought towards figuring out better ways to work.

Inspiration

A few weeks ago, at the end of January 2020 I found a post titled Building personal search infrastructure for your knowledge and code on Hacker News by karlicoss. This post really resonated with me because keeping track of all my notes and bookmarks was something that was contributing to my research and note taking workflow issues. I looked around a bit more on the beepb00p.xyz blog and found some other interesting articles as well, like:

  • What data I collect on myself and why?
  • How to cope with having fleshy human brain
  • How to annotate literally everything

I've long been interested in reading about other peoples setups, everything from uses this to r/battlestations, or the occasionally hilarious rate my setup posts. So, when I found the posts above on beepb00p.xyz I was really intrigued by the level of detail the author goes into while describing their personal knowledge management setup, and discussing the many GitHub repositories they've authored to enable their personal knowledge management solutions.

At the same time, I had noticed quite a few posts about research, note taking, and knowledge management setups on Hacker News recently (others have observed this as well). Here are a few of them:

  • How to Take Smart Notes
  • Managing my personal knowledge base
  • Nototo - Build a unified mental map of notes
  • Roam: Why I Love It and How I Use It
  • Ask HN: How to Take Good Notes

There are definitely quite a few more posts like the ones above, but having read a lot these posts and the comments on HN, the big theme that emerged for me was that there are a lot of people using Emacs and Org mode as their all-in-one notetaking and task tracking system (not to mention text editor, and operating system).

Having read these posts, and found the beepb00p.xyz blog; as I was reading beepb00p, I found the beepb00p site page where the author goes into more detail about how the blog is setup, and even linked to the github repository for the site. This is where it all came together for me. I realized that the beepb00p blog was a static site built on top of Org mode files. I like to remember this as my mind blowing moment 🤯. This was exactly the kind of setup I wanted for myself:

  • A note taking tool that was free, and hackable.
  • The ability to combine my notes with my todo list, and time tracking.
  • A built in way to share my notes online, and incorporate them into my blog.
  • Limitless keyboard shortcuts to weave together all of my note taking and organizational notes and tasks.
  • A tool that would give me complete control over my data.

I could finally see why so many people are using Emacs and Org mode.

The only problem was I'd never used Emacs, or Org mode.

I ended up returning and reading nearly all of the articles on beepb00p.xyz over the course of the past month. I highly recommend checking it out if you're into this kind of stuff. I also spent a lot of time thinking about the content and the blog itself over the past month. There are a lot of details that I really liked, and have tried to reproduce here in my own blog. I'm planning to write another more detailed post about reasoning behind various elements of how I've configured this blog, but for now I've begun to detail the way I'm running the blog in my Blog Setup Notes.

Roadmap #wip

I've started learning and using Emacs and Org mode in the hope that they can become foundational elements of my v0.2 #pkm system. So far, I'm learning a lot and continually impressed with all the work that people have done to make awesome Emacs plugins. However, it hasn't been without a lot of hard work and head-scratching trying to figure out how to fix a lot of minor issues. I'm not surprised by how many posts I've found about the pain and despair of configuring emacs, or "A Life Configuring Emacs" by Musa Al-hassy that clocks in at a 160 minute read! Trying to get everything just right seems to be a never-ending battle, but I'm getting pretty close to feeling like I have things in a state I can begin to work more with.

Luckily, I didn't see or focus on any of the more negative posts up front, or I may have dismissed even pursuing these tools in the first place. I tried Emacs briefly in the past, but could never get over the default keybindings with so many Ctrl (C-) and ESC (M-) key chords. And, when I first found Vim, the search for an editor ended for me.

Anway, in case you're considering exploring Emacs/Org mode, here are a few great YouTube videos I watched that gave me a better idea of what I could do with Emacs and Org mode, and how to get from Vim to Org mode:

  • Getting Started with Org Mode by Harry Schwartz
  • Evil Mode: Or, How I Learned to Stop Worrying and Love Emacs by Aaron Bieber
  • Emacs Org-mode - a system for note-taking and project planning by Carsten Dominik, the creator of Org mode

While I'm taking the time to rethink my #pkm setup, I figured I should also try to take a step back and get some more perspective from others who've spent a lot of time thinking about these things. I'm currently reading two great books that are already helping me think more deeply about how and what I capture, and how to organize things:

  • How to Take Smart Notes by Sönke Ahrens
  • Getting Things Done by David Allen

I'm going to continue reading these two books and taking notes, and will share an update once I've figured out how I might incorporate them into my system.

I'm still figuring out how to use Emacs and Org mode effectively. I've also found and adopted Spacemacs (develop branch), which really resonated with me once I was deep in keybinding hell trying to figure out all the minor differences between Vim and Emacs Evil Mode. When I finally stumbled back onto the Spacemacs homepage (probably from somewhere on Reddit, looking for keybinding advice), the Spacemacs "mantra" felt like a message from god:

Spacemacs is a new way to experience Emacs – a sophisticated and polished set-up focused on ergnomics, mnemonics and consistency.

– Spacemacs homepage

As far as I can tell, Spacemacs is definitely not without its own issues (e.g. I'm on the develop branch because of plugin issues in master). However, by combining ergonomics, mnemonics, and consistency, with the discoverability of the SPC keybinding menuing system, Spacemacs seems like a great Emacs starter kit that is pretty exciting for me (especially considering that there seems to be some pretty active development taking place in the develop branch).

I've also found countless great Emacs and Org related packages, so I started an Emacs Packages TODO List to keep track of everything I want to try. I haven't had a chance to try them all out yet.

I'm trying to keep my Emacs config as vanilla as possible given that I know so little about how congifuration changes might cause problems.

Next

Now that I've embarked on this journey to rethink my approach to personal knowledge management, I'm also trying to better understand the current state of #pkm tooling and solutions in the process. I've already begun testing some other tools in this space like Notion, Roam, Todoist and Remember the Milk, and I'm planning on sharing my thoughts on some of them in upcoming posts. If you have any other suggestions for great tools worth looking at, please let me konw!

I'm looking forward to sharing my notes and ideas on this blog, and hoping that I might be able to find some other like-minded people to have good discussions with.
